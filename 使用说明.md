# 无限画布使用说明

## 项目定位

无限画布是一个面向图片、视频、音频和文本创作的浏览器工作台。它把 AI 生成、节点编排、提示词、素材和本地 Agent 放在同一个界面中，但不提供项目自己的用户账号、计费后台或云端数据库。

## 页面说明

| 页面 | 路径 | 用途 |
| --- | --- | --- |
| 首页 | `/` | 进入主要创作模块 |
| 图片创作 | `/image` | 文生图、图生图、参考图编辑和生成记录 |
| 视频创作 | `/video` | 视频生成、异步任务和生成记录 |
| 我的素材 | `/assets` | 管理文本、图片和视频素材 |
| 提示词库 | `/prompts` | 搜索、筛选和收藏第三方提示词 |
| 我的画布 | `/canvas` | 创建、导入、导出和管理画布项目 |
| 画布项目 | `/canvas/:id` | 节点编排、AI 生成、助手与本地 Agent |

## 支持的画布节点

- 文本节点：保存提示词、文案、笔记和 AI 文本结果。
- 图片节点：保存上传图片、生成图片和素材图片。
- 视频节点：保存上传或生成的视频并提供原生播放。
- 音频节点：保存上传或生成的音频，也可作为生成参考。
- 生成配置节点：组合上游输入，配置模型、模式、尺寸、数量和输入顺序。

节点之间的连线表示上下游输入关系。生成配置节点会读取连接到它的文本、图片、视频和音频，不代表后台任务依赖或多人协作关系。

## AI 接口说明

Base URL 只允许使用 `https://www.aiba.hk`。

每个渠道可以选择 OpenAI 或 Gemini 调用格式，并维护自己的 API Key 和模型列表。

OpenAI 风格能力主要使用：

- `GET /v1/models`
- `POST /v1/images/generations`
- `POST /v1/images/edits`
- `POST /v1/responses`
- `POST /v1/audio/speech`
- `POST /v1/videos`
- `GET /v1/videos/{id}`
- `GET /v1/videos/{id}/content`

Gemini 格式当前用于模型列表、文本问答、在线 Agent 工具调用、基础生图和图生图。音频和视频仍需要兼容的 OpenAI 风格渠道。

Seedance 2.0 需要固定接口站点代理兼容火山方舟 Agent Plan。接口站点是否支持具体模型、参考媒体、参数和计费规则，以接口站点实际实现为准。

## 数据保存位置

| 数据 | 默认位置 | 说明 |
| --- | --- | --- |
| AI 配置和 API Key | 浏览器本地 | 由前端读取并直连固定接口站点 |
| 画布项目 | IndexedDB | 按项目持久化，包含节点、连线、会话和视口 |
| 图片文件 | IndexedDB | 以 Blob 和 `storageKey` 保存 |
| 视频及音频文件 | IndexedDB | 以 Blob 和 `storageKey` 保存 |
| 我的素材 | IndexedDB | 元数据与媒体文件分开保存 |
| 生成记录 | IndexedDB | 图片和视频记录分域保存 |
| 提示词缓存 | IndexedDB | 从 GitHub 来源拉取后的本地缓存 |
| WebDAV 配置 | 浏览器本地 | 同步时由浏览器直连远端 |

以下操作可能导致本地数据丢失：

- 清除该站点的浏览器数据。
- 使用无痕窗口并关闭全部窗口。
- 浏览器存储空间不足或被系统回收。
- 切换域名、协议或端口后继续使用，因为浏览器会视为不同站点。
- 浏览器配置文件损坏或被删除。

重要内容应使用 ZIP 导出和 WebDAV 双重备份。

## WebDAV 说明

WebDAV 用于跨设备同步，不是云端实时协作：

- 同步画布、素材、图片工作台和视频工作台数据。
- 同步关联图片、视频和音频 Blob。
- 根据记录 ID 和更新时间合并本地与远端数据。
- 使用删除墓碑传播删除状态。
- 写回远端前会重新合并最新本地数据，减少同步期间编辑被覆盖。

WebDAV 服务必须支持 HTTPS、认证、所需方法和浏览器 CORS。多设备同时编辑同一记录仍可能产生以最后更新时间为准的结果。

## 导入导出说明

- 画布 ZIP 和素材 ZIP 都包含结构化 JSON、媒体清单及媒体文件。
- 导入过程会限制压缩包大小、文件数和解压体积，并校验路径和媒体一致性。
- 导出包适合备份和迁移，不是稳定的第三方插件协议。
- 项目仍在开发阶段，未来格式可能直接调整，不保证旧数据永久兼容。

## 本地 Agent 说明

Canvas Agent 是可选的本机辅助服务：

- 连接网页画布与 Codex、Claude Code MCP 客户端。
- 默认监听 `127.0.0.1:17371`。
- 网页端拒绝非回环 Agent 地址。
- token 通过 URL fragment 或手动输入传递，不进入静态站请求日志。
- Agent 会限制允许连接的网页 Origin。
- MCP 或侧边栏提出的画布写操作仍由网页确认。

本地 Agent 不保存或代理网页中的 AI API Key。画布生成仍使用右上角配置的固定接口站点。

## 对外开放和多用户

静态前端可以供多名用户同时访问，因为每位用户的数据主要隔离在各自浏览器中，静态服务器也不承担 AI 推理。但需要理解以下边界：

- 项目本身没有账号体系和服务端用户空间。
- 不同浏览器之间不会自动看到同一份数据。
- 账号、计费、配额、审计、风控和权限隔离应由固定 AI 接口站点处理。
- API Key 会保存在用户浏览器并由浏览器直接发送给接口站点。
- 若需要团队共享项目、服务端审计或组织权限，需要另行开发后端能力。

## 浏览器和网络要求

- 推荐桌面端最新版 Chrome 或 Edge。
- 移动端可以访问，但复杂画布触控体验尚未系统完善。
- 页面和外部服务应使用 HTTPS。
- 固定接口站点、WebDAV 和媒体 URL 需要正确配置 CORS。
- 企业代理、浏览器扩展或严格防火墙可能阻止 GitHub 提示词源、本地 Agent 或媒体下载。

## 安全注意事项

- 只在可信设备和可信域名中填写 API Key。
- 不要把包含 `#apiKey=` 的完整链接发送给其他人。
- 不要把本地 Canvas Agent 监听地址改为公网地址。
- 对未知画布包和素材包保持谨慎，导入后检查节点和媒体内容。
- 对外站点应启用 HTTPS、安全响应头、访问日志轮换和基础可用性监控。
- 静态站服务器不会收到 AI API Key，但浏览器扩展、恶意脚本和终端设备仍可能读取页面数据。

## 常见问题

### 为什么换浏览器后看不到画布

数据保存在原浏览器对应域名的 IndexedDB 中。需要在原环境导出 ZIP，或先完成 WebDAV 同步再到新环境同步。

### 为什么模型列表为空

检查 API Key、接口格式和固定接口站点是否实现模型列表接口。也可以手动填写接口站点明确支持的模型名。

### 为什么生成成功后媒体后来失效

远程 URL 可能有有效期，且 CORS 可能阻止前端下载为本地 Blob。重要媒体应及时下载或确认已成功写入本地存储和备份包。

### 为什么 WebDAV 测试失败

常见原因是证书无效、账号错误、目录无权限、CORS 缺少所需方法或 HTTPS 页面访问 HTTP WebDAV 被浏览器拦截。

### 为什么线上页面连接不到 Canvas Agent

Agent 必须运行在打开浏览器的同一台电脑上，并使用回环地址。首次连接还会绑定页面 Origin，切换域名后可能需要重新授权或清理 Agent Origin 配置。
